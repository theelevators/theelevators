The origin

I wanted internet at sixteen. Couldn't afford it. So I built an antenna to find the signal instead.

Being too broke for internet, pushed me to build a high-gain antenna out of an old satellite dish and a USB WiFi adapter just to find a signal. That's where I learned my first real lesson: if you want a system to work, you have to understand exactly how its rules are built and how to enforce them.

I came to the United States a couple years before my first real job. Barely spoke English. Couldn't afford college. So I took the only door open to me: a part-time counter job at an auto parts store. No plan. Just a willingness to figure it out, the same instinct that built the antenna.

That instinct has found bigger systems to work on ever since. Today it's grown into a decade of professional architecture, the last several years in the high-stakes world of pharmacy and clinical data. A mistake here isn't just a bug. It's a liability. I don't believe in moving fast and breaking things. I build systems with a constitution: structural rules that ensure data is verified, signed, and persistent before it ever moves.

Right now I'm building at the frontier of agentic orchestration. Engines that let AI organize and reason about information through context compression, moving past simple databases into something closer to a nervous system. The kid who built an antenna to find a signal is still here. He just found a bigger problem to solve.

Tools · Code Intelligence
ACC
Adaptive Codec Context: a living physics engine for codebases
ACC transforms codebases into a dimensional semantic graph where every entity is measured across four dimensions: Stability, Logic, Friction, and Autonomy. The AVEC vector. No ML, no black box. Pure observable metrics: git churn, cyclomatic complexity, dependency edges, contributor count.

Query your codebase like a system. Find fragile chokepoints, run impact analysis before refactoring, discover similar implementations by dimensional profile. Exposed via VS Code extension, Neovim plugin, CLI, and MCP server.

Tools · AI Memory
STTP-MCP
Spatio-Temporal Transfer Protocol: persistent AI memory
Every AI session starts from zero. STTP encodes the meaning of a conversation directly as a typed intermediate representation with confidence-weighted fields and AVEC state vectors that describe the cognitive geometry of the conversation at the moment of compression.

An MCP server exposes this as five tools any model can call: store context at a milestone, retrieve it by attractor resonance, calibrate session state, list stored nodes, and shift reasoning posture intentionally. Model-agnostic by design. One session stores, another retrieves, and the state is self-sufficient enough to reconstruct from.
